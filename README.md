# Python Quiz — Single Page + Telegram forwarding

Короткая инструкция по запуску локально:

- Склонируйте/расположите файлы в папке (это уже в `docs/`).
- Перейдите в папку `server` и установите зависимости:

```powershell
cd server
npm install
```

- Создайте `.env` на основе `.env.example` и укажите реальные `BOT_TOKEN` и `CHAT_ID`.

Если вы хотите развернуть сайт как единый проект (статический фронтенд + serverless-функция), рекомендуется использовать Vercel или Netlify.

1) Vercel (рекомендуемый):

	- Перенесите репозиторий на GitHub.
	- Создайте проект на https://vercel.com и импортируйте репозиторий.
	- В настройках проекта добавьте переменные окружения `BOT_TOKEN` и `CHAT_ID` (не добавляйте их в код в репозиторий).
	- Vercel автоматически развернёт фронтенд (корень) и `api/submit.js` как serverless-функцию на `/api/submit`.
	- После деплоя откройте публичную ссылку — форма будет отправлять данные на `/api/submit`, и бот пришлёт ответы в Telegram.

2) Netlify: аналогично — подключите репозиторий, добавьте environment variables и используйте Netlify Functions (путь/подключение может отличаться).

3) Если вы хотите запускать локально (или предпочли Express):

```powershell
cd server
npm install
npm start
```

и откройте `http://localhost:3000/`.

Важно по безопасности: храните `BOT_TOKEN` и `CHAT_ID` в переменных окружения провайдера хостинга (Vercel/Netlify), а не в публичном репозитории.

- Запустите сервер:

```powershell
npm start
```

Откройте в браузере `http://localhost:3000/` — вы увидите страницу теста.

Как это работает:
- Форма отправляет POST `/submit` с JSON телом.
- Сервер собирает текст и пересылает его через Telegram Bot API на указанный `CHAT_ID`.

Примечание по безопасности: не храните публично `BOT_TOKEN`. Для упрощённой демонстрации токен используется на сервере, не в клиенте.
